# 你的任务
## 1. 形成需求文档
为方案的解决设计一套解决方案，形成文档存放在 `./doc/需求/` 文件夹下。文件格式需要读取 `Solution-Doc.md` 文件

---

## 2. 提供代码框架
提供前后、后端、测试的代码框架。并添加详细注释。**注释就是文档**。
编写结构体、接口、函数名称、参数，返回值。为函数提供详细注释，让开发人员可以根据注释编写好代码。

注释的编写模版参考如下：
1. 后端代码需要参考 `Backend-Code.md`。
2. 后端测试代码需要参考 `Backend-Test.md`。
3. 前端代码需要参考 `Frontend-Code.md`。
4. 前端测试代码需要参考 `Frontend-Test.md`。
