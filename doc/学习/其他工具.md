# curl 工具

curl 是一个命令行 HTTP 客户端，用于发送请求和调试接口，是后端开发最常用的测试工具之一。

## 基本语法

```sh
curl [选项] [URL]
```

## 常用参数速查

| 参数 | 含义 | 示例 |
|------|------|------|
| `-X` | 指定请求方法 | `-X POST`、`-X PUT`、`-X DELETE` |
| `-H` | 添加请求头 | `-H "Content-Type: application/json"` |
| `-d` | 请求体数据 | `-d '{"name":"test"}'` |
| `-i` | 显示响应头 | `-i` |
| `-v` | 显示详细通信过程 | `-v` |
| `-s` | 静默模式（不显示进度） | `-s` |
| `-o` | 输出到文件 | `-o response.json` |
| `-k` | 跳过 SSL 证书验证 | `-k` |
| `-L` | 跟随重定向 | `-L` |
| `-w` | 自定义输出格式 | `-w "\n%{http_code}\n"` |

## GET 请求

```sh
# 最简单的 GET 请求
curl http://localhost:8082/api/v3/admin/orders

# 带查询参数
curl "http://localhost:8082/api/v2/orders?status=1&page=1"

# 带请求头（如 Authorization）
curl -H "Authorization: Bearer eyJhbGciOiJIUzI1NiJ9..." \
  http://localhost:8082/api/v3/admin/orders

# 查看响应头（-i）
curl -i http://localhost:8082/api/v3/admin/orders

# 查看完整通信过程（-v，调试用）
curl -v http://localhost:8082/api/v3/admin/orders
```

## POST 请求

```sh
# 发送 JSON 数据（最常用）
curl -X POST \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"123456"}' \
  http://localhost:8082/api/v3/admin/login

# 发送表单数据
curl -X POST \
  -d "username=admin&password=123456" \
  http://localhost:8082/api/v3/admin/login

# 从文件读取请求体
curl -X POST \
  -H "Content-Type: application/json" \
  -d @request.json \
  http://localhost:8082/api/v3/admin/login
```

## PUT / DELETE 请求

```sh
# PUT 请求（更新资源）
curl -X PUT \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <token>" \
  -d '{"status":8}' \
  http://localhost:8082/api/v3/admin/review/1/approve

# DELETE 请求
curl -X DELETE \
  -H "Authorization: Bearer <token>" \
  http://localhost:8082/api/v2/orders/12345
```

## 常见调试场景

### 1. 只看 HTTP 状态码

```sh
# -s 静默模式 + -w 输出状态码 + -o 丢弃响应体
curl -s -o /dev/null -w "%{http_code}" http://localhost:8082/api/v3/admin/orders
# 输出：200
```

### 2. 测试接口响应时间

```sh
# 输出：响应时间（秒）
curl -s -o /dev/null -w "time_total: %{time_total}s\n" http://localhost:8082/api/v3/admin/orders
# 输出：time_total: 0.035s
```

### 3. 下载文件

```sh
# -o 指定保存文件名
curl -o avatar.png https://example.com/avatar.png

# -O 使用远程文件名保存
curl -O https://example.com/avatar.png
```

### 4. 跟随重定向

```sh
# 默认 curl 不跟随重定向（301/302），加 -L 跟随
curl -L http://example.com/old-path
```

### 5. 发送 Cookie

```sh
# 方式一：直接在请求头中设置
curl -H "Cookie: session_id=abc123" http://localhost:8082/api/v3/admin/orders

# 方式二：使用 -b 参数
curl -b "session_id=abc123" http://localhost:8082/api/v3/admin/orders

# 方式三：从文件读取 Cookie
curl -b cookies.txt http://localhost:8082/api/v3/admin/orders
```

### 6. 保存并使用 Cookie（模拟登录态）

```sh
# 登录时保存 Cookie 到文件（-c）
curl -c cookies.txt \
  -X POST \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"123456"}' \
  http://localhost:8082/api/v3/admin/login

# 后续请求携带 Cookie（-b）
curl -b cookies.txt http://localhost:8082/api/v3/admin/orders
```

### 7. 调试 HTTPS 接口

```sh
# 跳过 SSL 证书验证（测试环境常用）
curl -k https://localhost:8443/api/v3/admin/orders
```

### 8. 格式化 JSON 输出

```sh
# 配合 python 或 jq 格式化
curl -s http://localhost:8082/api/v3/admin/orders | python3 -m json.tool

# 使用 jq（更强大，需安装）
curl -s http://localhost:8082/api/v3/admin/orders | jq .
```

## 本项目接口测试示例

```sh
# 1. 管理员登录，获取 token
TOKEN=$(curl -s -X POST \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"123456"}' \
  http://localhost:8082/api/v3/admin/login | jq -r '.data.token')

echo "Token: $TOKEN"

# 2. 使用 token 获取订单列表
curl -s -H "Authorization: Bearer $TOKEN" \
  http://localhost:8082/api/v3/admin/orders | python3 -m json.tool

# 3. 审核通过
curl -s -X POST \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"action":"approve"}' \
  http://localhost:8082/api/v3/admin/review/1/approve | python3 -m json.tool
```
