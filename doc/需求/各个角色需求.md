# 普通用户
1. 点击预约按钮
2. 跳转到预约页面
3. 查看已被预约时间段
4. 选择预约时间段
5. 填写预约申请，提交预约申请
6. 撤销预约申请

# 后台工作人员
1. 登入后台网页
2. 查看预约申请
3. 手动允许或取消，允许的同时填写密码，然后后台自动推送到普通用户手里

现在采取的架构是，nginx监听请求，如果是微信服务器发送的请求，发送到8080端口，如果是预约网页发送的请求，发送到8081端口，如果是管理员审核网页的请求，发送到8082端口。

目前项目的开发采用根据功能划分的一体式，采用go语言开发，cmd/api/v1文件夹下面会有多个主程序，分别监听8080，8081，8082。internal文件夹下划分了notification，auth，reserver，admin这几个功能。

notification处理信息消息流通，推送

auth处理用户信息管理，用户验证

reserve处理预约流程

admin处理审核流程
