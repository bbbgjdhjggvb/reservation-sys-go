```yaml
# 改前
command: --default-authentication-plugin=mysql_native_password --character-set-server=utf8mb4 --collation-server=utf8mb4_unicode_ci
# 改后

```

# 3. 跨域请求允许
## 问题描述
`swagger ui` 运行在 `localhost:8083`，而 `v2 API` 运行在 `localhost:8081`。两个程序运行在不同的端口，即出现 **跨域** 的现象。

浏览器会在发送 `POST` 请求之前，发送一个 `OPTIONS` 预检请求，但是我的 `v2` 服务没有配置 `CORS` 中间件来处理这个 `OPTIONS` 请求，所以会返回 404

## 解决方法
使用 `CORS` 中间件

# 404 网页无法访问报错原因排查

## 可能原因
### 1. 前端静态资源文件路径问题
```sh
<link rel="stylesheet" href="./static/css/style.css">
```
当用户访问 `http://domain.com/reserve?token=xxx` 时，浏览器会把 `./static/css/style.css` 解析为 `/reserve/static/css/style.css`。

这个路径不匹配 `nginx` 的静态资源位置

**解决方法**: 不要使用绝对路径，和nginx匹配
