# 微信菜单同步工具使用指南
## 概述

菜单同步工具是一个独立的命令行工具，用于将本地菜单配置同步到微信公众号，无需启动完整的后端服务。

---

## 🚀 快速开始

### 本地开发环境

```bash
# 方式1: 使用默认配置
go run cmd/tools/sync_menu.go

# 方式2: 指定本地配置
go run cmd/tools/sync_menu.go -config configs/config_sync_menu.local.yaml

# 方式3: 使用环境变量
CONFIG_PATH=configs/config_sync_menu.local.yaml go run cmd/tools/sync_menu.go
```

## 🔧 使用场景

### 场景 1: 本地开发测试菜单

```bash
# 1. 修改菜单配置
vim configs/menu.json

# 2. 同步到微信
go run cmd/tools/sync_menu.go -config configs/config_sync_menu.local.yaml

# 3. 重新关注公众号查看效果
```

### 场景 2: 测试环境更新菜单

```bash
# 1. SSH 到云服务器
ssh user@your-server-ip

# 2. 进入容器
docker exec -it reservation-v1 sh

# 3. 同步菜单
/app/v1 -tools sync-menu

# 或直接执行
docker exec reservation-v1 /app/v1 -tools sync-menu
```

### 场景 3: CI/CD 自动同步

```bash
# 在 CI/CD 流程中自动同步菜单
docker exec reservation-v1 /app/v1 -tools sync-menu
```

## 📝 菜单配置规范

### 文件位置

`configs/menu.json`

### 格式要求

```json
{
  "button": [
    {
      "type": "view",
      "name": "预约场地",
      "url": "https://your-domain.com/reserve"
    },
    {
      "name": "帮助",
      "sub_button": [
        {
          "type": "view",
          "name": "使用说明",
          "url": "https://your-domain.com/help"
        },
        {
          "type": "click",
          "name": "联系客服",
          "key": "contact_service"
        }
      ]
    }
  ]
}
```

### 限制说明

1. **一级菜单**: 最多 3 个，每个最多 4 个汉字
2. **二级菜单**: 每个一级菜单最多 5 个，每个最多 8 个汉字
3. **菜单类型**:
   - `click`: 点击推事件
   - `view`: 跳转 URL
   - `scancode_push`: 扫码推事件
   - `scancode_waitmsg`: 扫码带提示
   - `pic_sysphoto`: 系统拍照发图
   - `pic_photo_or_album`: 拍照或者相册发图
   - `pic_weixin`: 微信相册发图
   - `location_select`: 发送位置

## 🔄 配置优先级

工具按以下顺序查找配置文件：

1. **命令行参数**: `-config` 指定的路径
2. **环境变量**: `CONFIG_PATH`
3. **默认配置**: `configs/config_sync_menu.yaml`
4. **降级配置**: `configs/config_v1.yaml`

## ⚠️ 注意事项

### 1. 测试菜单生效

修改菜单后，测试方法：

- **方式1**: 取消关注后重新关注公众号
- **方式2**: 等待 24 小时自动刷新
- **方式3**: 使用微信公众平台的"菜单重试"功能

### 2. 菜单按钮 URL
为了鉴权，我们需要获取用户的 Openid，获取用户的 Openid 需要进行 OAuth2 授权。微信规定获取授权的重定位连接必须符合如下规范：
```sh
https://open.weixin.qq.com/connect/oauth2/authorize?appid=wx84d6833105361902&redirect_uri=https%3A%2F%2Fyour-domain.com%2Fapi%2Fv1%2Fauth%2Fcallback&response_type=code&scope=snsapi_base#wechat_redirect
```
需要我们填写的有：
1. appid
2. redirect_uri
3. scope(授权方式): `snsapi_base(静默授权，只能获取Openid)` 或 `snsapi_userinfo`
