# 环境变量
Docker Compose 在运行时，默认会自动寻找当前目录下的 `.env` 文件，
并从中加载环境变量。

所以，部署的第一步是创建 `.env` 文件。
```sh
cp .env.example .env
```

# 容器部署

1. 解压缩打包文件

```sh
tar -xzvf reservation-sys-prod.tar.gz
```

2. 加载镜像

```sh
docker load -i gateway.tar
docker load -i reservation.tar
docker load -i admin.tar
```

3. 启动容器

```sh
docker-compose -f docker-compose.prod.yaml up -d
```

# 停止系统

```sh
docker-compose -f docker-compose.prod.yaml down
```
