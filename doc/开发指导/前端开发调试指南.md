# 前端开发调试指南

## 前端框架介绍

pnpm monorepo(单体仓库) + Vue3 + TypeScript
- [pnpm 包管理器优势](https://pnpm.io/package-sources)

## 前置条件

- Node.js >= 20
- pnpm（如未安装：`npm install -g pnpm`）

## 项目结构

```
frontend/
├── package.json              # 各个仓库运行，构建命令的配置
├── pnpm-workspace.yaml       # 生命 mono 子包的位置
├── tsconfig.base.json        # 所有子包共享的 TypeScript 配置
└── packages/
    ├── shared/               # 共享类型定义（@reservation/shared）
    │   └── src/types.ts
    ├── reservation/           # 用户预约端 SPA
    │   ├── vite.config.ts     # 端口 5173，base: /
    │   └── src/
    │       ├── views/         # 页面：ReserveView, MyOrdersView
    │       ├── components/    # 组件：CalendarGrid, ReserveForm 等
    │       ├── composables/   # useCalendar, useToast
    │       ├── stores/        # Pinia: auth
    │       ├── api/           # 后端 API 调用封装
    │       └── utils/         # 日期工具函数
    └── admin/                 # 管理员后台 SPA
        ├── vite.config.ts     # 端口 5174，base: /admin/
        └── src/
            ├── views/         # 页面：LoginView, DashboardView
            ├── components/    # 组件：AdminOrderCard, OrderDetailModal 等
            ├── composables/   # useAdminOrders
            ├── stores/        # Pinia: admin
            └── api/           # 后端 API 调用封装
```

## 安装依赖

```sh
cd frontend
pnpm install
```

## 启动开发服务器

```sh
cd frontend

# 预约端 — http://localhost:5173/
pnpm dev:reservation

# 管理后台 — http://localhost:5174/admin/
pnpm dev:admin
```

两个命令可以在不同终端窗口中同时运行。

Vite 开发服务器支持 **HMR 热更新**，修改 Vue/TS/CSS 代码后浏览器会即时刷新，无需手动刷新。

## 关于 API 代理

开发服务器内置了 Vite proxy，会自动将 API 请求转发到本地 Go 后端：

| 前端请求路径 | 转发到 |
|---|---|
| `/api/reservation/*` | `http://localhost:8081` |
| `/api/admin/*` | `http://localhost:8082` |

因此在开发前端之前，需要先启动后端服务：

```sh
# 1. 启动 MySQL + Redis
docker-compose -f docker-compose.local.yaml up -d mysql redis

# 2. 启动 Go 服务（各开一个终端）
go run service/reservation/cmd/main.go
go run service/admin/cmd/main.go
```

> 预约端 API 需要认证 token，Go 服务启动后还需要启动 gateway（`go run service/gateway/cmd/main.go`）完成微信 OAuth 回调获取真实 token。本地测试也可以直接使用脚本生成 token：
> ```sh
> bash scripts/env.sh token
> # 或 go run tools/jwt/gen_token.go
> ```

## 预约端访问方式

预约端依赖用户 JWT token 进行认证。在浏览器中访问时需要在 URL 上附带 token：

```
http://localhost:5173/?token=<jwt_token>
```

如果没有 token，页面会显示"未授权访问"提示。

## 管理后台访问方式

管理后台通过用户名/密码登录获取 admin token，不需要在 URL 上传 token：

```
http://localhost:5174/admin/login
```

## 调试技巧

### Vue DevTools

推荐在浏览器中安装 [Vue DevTools 扩展](https://devtools.vuejs.org/)，可以：
- 查看组件树和组件状态
- 实时修改组件 data/props
- 查看 Pinia store 状态
- 追踪事件和路由

### 浏览器开发者工具

建议将 Chrome DevTools 的下列选项打开：
- **Network 面板** → 勾选 "Disable cache"（开发期间避免浏览器缓存）
- **Console 面板** → 查看 Vue 警告和 API 请求错误

### 模拟微信浏览器

预约端需要在微信内置浏览器中使用。在桌面端开发调试时可以通过以下方式模拟：

1. Chrome DevTools → Toggle device toolbar（Ctrl+Shift+M）→ 选择 iPhone/Android 设备
2. 安装微信开发者工具，用于测试微信 JSSDK 相关功能

> 样式已兼容微信浏览器：`user-select: none` 防止长按选中，`-webkit-tap-highlight-color: transparent` 去除点击高亮。

### 移动端真机调试

如果需要在手机上查看效果：

```sh
cd frontend
pnpm --filter @reservation/app dev --host
```

Vite 会输出一个局域网地址（如 `http://192.168.x.x:5173/`），在手机浏览器中访问即可。注意手机和开发机需在同一局域网。

## 构建生产版本

```sh
cd frontend
pnpm build:all
```

构建产物统一输出到项目根目录下的 `dist/`：

```
dist/
├── reservation/    # 预约端
│   ├── index.html
│   └── assets/
└── admin/          # 管理后台
    ├── index.html
    └── assets/
```

构建后的文件由 nginx 直接提供，通过两个独立 volume 挂载到容器内不同路径。实现了前后端完全分离（前端源码在 `frontend/`，构建产物在 `dist/`，Go 服务在 `service/`）。

## 常见问题

### pnpm 未找到

```sh
npm install -g pnpm
```

### 依赖安装报错

某些包的构建脚本（如 esbuild）需要批准：

```sh
cd frontend
pnpm approve-builds esbuild vue-demi
pnpm install
```

### 端口被占用

预约端默认 `5173`，管理后台默认 `5174`。如果端口被占用，可以在对应的 `vite.config.ts` 中修改 `server.port`。

### 修改代码后页面未更新

1. 确认 dev server 终端没有报错
2. 如果有语法错误，Vite 会在页面上显示错误遮罩层
3. 少数情况下 HMR 可能失效，手动刷新浏览器即可

### API 请求 401

- 预约端：检查 URL 上的 `?token=` 是否有效，或 localStorage 中是否有 `auth_token`
- 管理后台：检查是否已登录，或 token 是否已过期，重新登录即可
