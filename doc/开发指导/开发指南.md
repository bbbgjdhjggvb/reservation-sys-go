# 本地开发
## 指定 go 的开发版本
在云端部署环境中，go 的版本为1.24，本地开发时 go 版本需要保持一致。
运行 `go mod tidy` go 的工具链条会自动将 go 的版本指定为最新的版本。

解决方法如下：

在 vsccode IDE 中添加`.vscode/settings.json` 文件
```json
{
    // 1. 强制 VS Code 的 Go 插件使用指定的本地工具链行为
    "go.toolsEnvVars": {
        "GOTOOLCHAIN": "go1.24.0" 
    },

    // 2. 强制 VS Code 内置终端也使用相同的环境变量
    // 这样你在 VS Code 终端里直接敲 go mod tidy 也不会乱改版本
    "terminal.integrated.env.linux": {
        "GOTOOLCHAIN": "go1.24.0"
    },

    // 3. （可选）如果你在系统里安装了多个 Go 版本（比如通过 ~/sdk/go1.24.0 安装）
    // 你可以直接指定 Go 执行文件的绝对路径，让代码提示(gopls)直接使用它
    // "go.alternateTools": {
    //     "go": "/home/你的用户名/sdk/go1.24.0/bin/go"
    // }
}
```

# Docker
## 不能在本地构建 Docker 镜像
``` sh
# 将代码从本地同步云端服务器上
git push ssh://root@106.52.23.213/root/git/reservation_sys_go.git master

# 运行脚本构建 Docker 镜像，并进行部署
sh deploy-local.sh
```

## 在本地进行 Docker 镜像构建然后上传镜像
需要有代理环境

```sh
# 构建镜像
docker build -t reservation-sys:test .

# 导出镜像
docker save -o reservation-sys.tar reservation-sys:test

tar -czf reservation-sys-test.tar.gz \
    reservation-sys.tar \
    docker-compose.test.yaml \
    configs/config_v1.yaml \
    configs/config_v2.yaml \
    deploy/ \
    internal/reservation/frontend/

# 上传到服务器
scp reservation-sys-test.tar.gz root@106.52.23.213:/root/reservation-sys/

# 在服务器上解压缩
tar -xzvf reservation-sys-test.tar.gz

# 加载镜像到docker中
docker load -i reservation-sys.tar

# 在本地开发测试
1. 在本地 docker 启动 redis, mysql 服务
```sh
# 启动 mysql, redis 的 Docker 服务
# 或者 bash script/env.sh up
docker-compose -f docker-compose.local.yaml up -d mysql redis
```

2. 使用脚本生成 token: openid 在 gen_token.go文件中设置
```sh
# openid 默认是 test_openid_local_001
# 或者 bash script/env.sh token
go run /cmd/tool/jwt/gen_token.go openid
```

3. 启动服务
```sh
# 或者 bash script/env.sh serve
go run cmd/api/v2/main.go --config configs/config_v2.debug.yaml

```

4. 运行请求测试脚本
```sh
bash script/test-api.sh
```

5. 清楚服务，mysql，redis
```sh
# 脚本只停止服务程序
bash script/env.sh down

# 卸载 mysql, reids 容器
docker-compose -f docker-compose.local.yaml down mysql redis
```
