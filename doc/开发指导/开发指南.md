# 本地开发
## 指定 go 的开发版本
在云端部署环境中，go 的版本为1.24，本地开发时 go 版本需要保持一致。
运行 `go mod tidy` go 的工具链条会自动将 go 的版本指定为最新的版本。

解决方法如下：

在 vsccode IDE 中添加`.vscode/settings.json` 文件
```json
{
    // 1. 强制 VS Code 的 Go 插件使用指定的本地工具链行为
    "go.toolsEnvVars": {
        "GOTOOLCHAIN": "go1.24.0" 
    },

    // 2. 强制 VS Code 内置终端也使用相同的环境变量
    // 这样你在 VS Code 终端里直接敲 go mod tidy 也不会乱改版本
    "terminal.integrated.env.linux": {
        "GOTOOLCHAIN": "go1.24.0"
    }
}
```

# Docker 镜像构建
## 不能在本地构建 Docker 镜像
``` sh
# 将代码从本地同步云端服务器上
git push ssh://root@106.52.23.213/root/git/reservation_sys_go.git master

# 运行脚本构建 Docker 镜像，并进行部署
sh ./scripts/build.sh
```

---

## 在本地进行 Docker 镜像构建然后上传镜像
需要有代理环境
```sh
bash ./scripts/build.sh

# 将打包好的文件上传到服务器
scp reservation-sys-test.tar.gz root@106.52.23.213:/root/reservation-sys/

# 在服务器上进行解压缩
tar -xzvf reservation-sys-test.tar.gz
```

# 在本地开发测试
## 1. 在本地 docker 启动 redis, mysql 服务
```sh
# 启动 mysql, redis 的 Docker 服务
docker-compose -f docker-compose.local.yaml up -d mysql redis
```

## 2. 使用脚本生成 token
```sh
# openid 默认是 test_openid_local_001
# 或者 bash script/env.sh token
go run tools/jwt/gen_token.go
```

## 3. 启动服务
```sh
go run service/gateway/cmd/main.go 
go run service/reservation/cmd/main.go
go run service/admin/cmd/main.go
```

## 4. 启动 Swagger 文档服务
```sh
go run service/swagger/cmd/main.go
# 访问 http://localhost:8083/swagger/index.html
```

## 5. 运行请求测试脚本
```sh
bash script/test-api.sh
```

## 6. 停止服务，mysql，redis
```sh
# 卸载 mysql, reids 容器
docker-compose -f docker-compose.local.yaml down mysql redis
```
