# 开发流程
需求 -> 文档 -> 测试 -> 开发

核心: 文档、测试驱动。
先有文档，测试，再有开发。

---

# 文档编写

## 目的
1. 记录开发过程
2. 理清开发思路
3. 记录命令工具

---

## 分类
1. 开发文档: 需求 -> 设计 -> 代码实现
    1. 架构
    2. 模块
2. 测试文档: 设计测试用例
3. 工具文档: 记录命令行工具的使用
4. 学习文档: 记录一些组件，某些复杂工具的运行原理

---

# 注释要求
1. 每个函数需要在开头进行注释介绍
    1. 函数的作用，参数，返回值
2. 复杂的逻辑或者关键步骤需要添加代码注释
3. API 要有 swagger 注解
4. 数据库操作代码需要写对应 SQL
